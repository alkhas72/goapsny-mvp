# DESIGN.md — GoApsny (Доступная Абхазия)

Единый источник дизайн-правил проекта. Кладётся в корень репозитория. Все агенты (Claude Code, Codex, Antigravity, Lovable) читают ЭТОТ файл, чтобы рисовать один и тот же GoApsny, а не четыре разных. Этот файл — про интерфейс продукта (открытый веб). Он НЕ описывает Bitrix, MCP-оркестрацию или внешние контуры — только то, что видит пользователь.

**Статус:** рабочий черновик. Открытые пункты помечены `[TBD]`.

## 1. Характер

Тёплый · надёжный · спокойный · лаконичный.

Мы не хотим удивлять. Цель — интерфейс, которого не замечаешь, потому что уже нашёл то, что искал. Для утилитарного продукта незаметность интерфейса — высшая форма дизайна, а не отсутствие дизайна.

**Принцип пути:** максимально короткий путь. Если задачу можно решить в один клик — один клик. Второй клик добавляем только если этого требует логика, никогда «для красоты» или «чтобы выглядело солиднее».

**Антипримеры (чем GoApsny быть НЕ должен):**

- не игривый, не «кислотный», не трендово-кричащий;
- не сухая корпоративная холодность;
- не перегруженный информацией; не «мини-сайт» — это карта.

## 2. Среда и платформа

- Основной продукт — открытый веб, mobile-first. Открывается по ссылке в любом браузере. Без установки, без Telegram, без VPN на входе.
- Проектируем от мобильного экрана вверх. Десктоп — адаптация, не наоборот.
- Telegram / Instagram / Facebook — внешние двери (сообщество, новости), а НЕ среда приложения. В интерфейсе они присутствуют только как ссылки наружу.
- Целевые вьюпорты для проверки: 360 · 390 · 768 · 1024 · 1440 px.

## 3. Доступность — не обсуждается (Ось 1)

Приложение о доступности обязано быть доступным само, иначе оно себя опровергает. Это правило стоит ПЕРЕД генерацией любого экрана, а не после.

**Цель:** WCAG 2.2 AA как база. `[TBD]` — целиться ли в AAA по части критериев для флагманских экранов (карта).

**NEVER ship without:**

- Статус — всегда словом + цветом, никогда только цветом. Дальтонизм в аудитории карты для людей с инвалидностью гарантирован; пин без текстовой подписи для такого пользователя слеп.
- Контраст AA в обеих темах (тёмная тема и текст на полупрозрачном фоне — зона повышенного риска, проверять отдельно).
- Зоны нажатия ≥ 44 × 44 px (был реальный баг с иконкой ниже нормы — знаем).
- Alt-тексты на всех значимых изображениях.
- Полная навигация с клавиатуры (Tab / Enter / Esc), видимый фокус-индикатор, осмысленный порядок фокуса.
- Корректные ARIA-роли и метки.
- Поддержка `prefers-reduced-motion` (движение отключаемо).

**Проверка:**

- WAVE (webaim) — «минутный» чекер. ВАЖНО: GoApsny — клиентский SPA, поэтому URL-режим WAVE видит только серверную оболочку до загрузки JS. Гонять расширением на живой вкладке, не через вставку URL.
- axe-core (Deque) — глубже. Lighthouse — с оговоркой (может показать 100% на недоступном сайте, не доверять слепо).
- Козырь проекта: живой тест с реальными людьми с разными формами инвалидности — сильнее любого сканера.

Источники: WCAG 2.2 (W3C), axe-core, Lighthouse, WAVE (WebAIM).

## 4. Цвет

### 4.1 Бренд

- Голубой (первичный): `#4BA4DD`
- Оранжевый (акцент): `#E2741C`

### 4.2 Светофор доступности (семантика — цвет ВСЕГДА со словом)

- Доступно — зелёный
- Частично — янтарный / жёлтый
- Недоступно — красный (не прячем красный)
- Ещё не оценено — серый

### 4.3 Специальный маркер

- Приставной пандус — крупная пурпурная точка `#7A5AF8` на нейтральном пине.

### 4.4 Правило

Цвет несёт смысл только у статусов и маркеров. Остальной интерфейс — спокойный нейтральный. Не расширять смысловую палитру ради украшения. Одна палитра на весь продукт, без плавания между тёплыми и холодными нейтралями в одном экране.

## 5. Темы

Две темы, обе обязаны проходить контраст AA:

- **День** — светлая, оранжево-янтарная тёплая гамма.
- **Ночь** — тёмная сине-голубая.

Off-black / off-white, не чистые `#000` / `#FFF`.

## 6. Типографика

**Требование к шрифту (жёсткое):**

- Гротеск (без засечек) — быстрее и чище читается в мобильном интерфейсе.
- Обязано полностью покрывать абхазский глиф-сет (Ԥ ԥ Ҭ ҭ Ҵ ҵ Ҷ ҷ Ҩ ҩ Ә ә Ҟ ҟ Ҽ ҿ и т.д.), плюс русскую и латинскую системы, согласованно.
- Приоритет читаемости — абхазский.
- Файлы шрифтов хранить локально, вместе с приложением. НЕ тянуть с внешнего CDN: при блокировках/VPN внешний шрифт обрежется и абхазский посыплется первым.

**Кандидаты (финал — после теста на реальных файлах):**

1. **Commissioner** — фаворит. Нейтральный, современный, спокойный гуманистический гротеск. Ложится в характер «не удивлять». ⚠️ абхазское покрытие проверить СТРОЖЕ всех (фаворита глаз прощает — не давать ему поблажки).
2. **PT Sans / PT Astra Sans** — самое надёжное расширенно-кириллическое покрытие (линейка ПараТайп делалась под письменности народов РФ). PT Astra — сестра с ещё более полным покрытием, под официальный документооборот; на Google Fonts её нет, проверять отдельно из файлов.
3. **Fira Sans** — тёплый дружелюбный гротеск, отличная экранная читаемость.
4. **Nunito** — мягкий, скруглённый, самый приветливый. ⚠️ у скруглённых чаще дыры в расширенной кириллице — проверять придирчивее всех.

**Критерий выбора:** не «какой красивее», а кто чище несёт абхазский. Тест — контрольная строка `Ԥсыҭбарада · аԥсшәа · Аҟәа` + ряд одиночных глифов: если буква выбивается из ряда по толщине/форме — глифа нет, шрифт отпадает. Победитель — тот, где абхазская строка так же цельна, как русская. `[TBD — финал после теста]`

## 7. Экраны и структура (слой логики)

Карта — всегда фон. Все панели — оверлеи поверх живой карты. Приложение = карта, а не мини-сайт: каждая «дверь» ведёт ко всей нужной инфо.

- **Приветствие** (показ один раз). Оверлей поверх карты. Легенда 5 строк (зелёный / жёлтый / красный / серый «ещё не оценено» + пурпурная точка = приставной пандус). Кнопка «ОК, поехали!».
- **Карта.** Шапка: слева переключатель языков `ab / ru / en` (приоритет абхазского), справа поиск + фильтр + бутерброд-меню. Нижняя полоса: центр — Личный кабинет, края — Instagram / Telegram / сайт.
- **Бутерброд** — 4 пункта: Сообщество → Telegram; Новости → Instagram; Заказать пандус → лендинг; Добавить локацию (+ мелкий «Вход для волонтёров»).
- **Фильтр** (вкладка у поиска). 12 категорий единым словарём, мультивыбор; после выбора — статус-этаж (Все / Доступные / Частично / Недоступные), тап по статусу применяет и схлопывает вкладку. У поиска — индикация выбранного (иконки-кружочки + счётчик + сброс). 12 категорий: Магазины, Еда и напитки, Транспорт, Достопримечательности, Культура, Гостиницы, Обучение, Госучреждения, Здоровье, Финансы, Спорт, Туалеты.
- **Карточка места** (bottom-sheet, без промежуточного попапа). Тап по пину → сразу карточка. Шапка: иконка категории в цвете статуса + название + категория подписью + крестик. Статус словом + цветом (для уровня 1 — НЕ кликабельный). Фото во всю ширину. Адрес. Индикатор пандуса (пурпурная точка). Строка «Проверено: [роль] · [когда]». Сворачиваемый список «Есть на: Яндекс ↗ / Google ↗ / Apple ↗ / OSM ↗» (Яндекс первым). «Сообщить об ошибке». Для заявки автора — янтарная плашка «На проверке у модератора. Видно только вам».

## 8. Уровни пользователей и путь

- Просмотр — всегда без регистрации. Захотел смотреть — смотришь всё.
- Барьер только на «добавить»: почта → код на ту же почту (без пароля). Рождает личный кабинет.
- **Уровень 1 (базовый):** смотрит свободно; добавляет через шаг 1.
- **Уровень 2 (аудитор / координатор):** аудит, триаж, модерация.
- Карточка ОДНА для всех — разница только в правах на заполнение, не в просмотре. Сложность спрятана от того, кому не нужна, раскрыта тому, кто с ней работает (прогрессивное раскрытие).

**Конвейер заполнения** — один поток, этапы привязаны к уровню, не к объекту:

- **Шаг 1 (уровень 1):** фото (до 5, обязательно — нет фото, нет добавления), категория (обязательно), название (обязательно), адрес опц., свёрнутый добровольный комментарий. Тихий анти-дубль «рядом уже есть» после установки пина.
- **Шаги 2–3 (аудитор):** обязательные поля + индикатор готовности этапа (защита от невнимательности; аудитор в поле, возможно слабое зрение → доступность инструмента аудитора тоже критична).
- **Шаг 4 (опционален, камерально):** кросс-ссылки OSM / Google / Apple / Яндекс + сайт/соцсети объекта как дополнительная верификация. Отделён от полевой работы; может вестись отдельными людьми с участием агентов (Bitrix-контур).

**Уровень 2 — состав инструмента аудитора:** блок «Инфраструктура доступности» (да/нет: приставной пандус, подъёмник/лифт, парковка ИНВ); замеры (ступеней, высота, ширина двери); «Краткое описание» (AI-копайлот) + «Заметки волонтёра» (свободный текст); категорированные доп-фото (Вход / Пандус / Туалет / Внутри). AI-копайлот черновит атрибуты по фото с % уверенности; источник правды — аудитор, AI-угаданное не публикуется как проверенное.

Блоки отображения ≠ этапы заполнения.

## 9. Границы (не размывать)

- Соцсети ВНУТРИ продукта — НЕТ. Сообщество живёт в Telegram, новости в Instagram.
- Публичных комментариев / рейтингов / лайков — НЕТ. Наша «звёздность» = статус доступности. Оценка доступности воспроизводима (норма пандуса либо соблюдена, либо нет), а не субъективна как отзыв.
- Геймификации — НЕТ (баллы / XP / лидерборды / бейджи убираем). Уровни как роли — оставляем.
- Данные хранить под теги OSM (`wheelchair`, `elevator`, `parking:disabled`) с самого начала. Экспорт в OSM/Google/Яндекс — отдельный Research-трек (у OSM строгие правила импорта); сейчас только заложить в модель данных.

## 10. Движение (последний слой, пока отложен)

Пока статика не собрана — не анимируем. При добавлении движения:

- анимации UI < 300 мс, кастомные кривые вместо дефолтных;
- вход из `scale(0.95)`, не из `scale(0)`;
- НЕ анимировать высокочастотные и клавиатурные действия;
- всё движение отключаемо через `prefers-reduced-motion`. Для утилитарной карты сдержанность в движении важнее богатства движения.
