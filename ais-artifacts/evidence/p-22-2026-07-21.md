### Код-ревью: синхронного `<script>` с telegram.org нет

- `index.html` — упоминание `telegram.org` только в HTML-комментарии (строки 11-17), без тега `<script>`. Подтверждено: `rg "telegram\.org|telegram-web-app" index.html public/` → 1 совпадение, в комментарии.
- `src/main.tsx:14-27` — bootstrap вызывает `ensureTelegramSdk()`, который внутри (`src/utils/telegramSdk.ts:48-52`) сначала проверяет `looksLikeTelegramLaunch(window.location)` (параметр `tgWebApp` в hash/search). Если запуск не из Telegram — SDK вообще не запрашивается. Если из Telegram — грузится с `async` и таймаутом 5с, после которого страница рисуется без него.
- `src/App.tsx:15-24` — детект через `window.Telegram?.WebApp?.initData`. Нет initData → `PublicApp`. Любая ошибка чтения → catch → тоже `PublicApp`.

Цепочка зависимостей от Telegram разорвана на трех уровнях: `<head>`, загрузка SDK, выбор компонента.
