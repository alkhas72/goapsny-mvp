## 1. Product purpose

Показывает людям с инвалидностью, какие места в Абхазии физически доступны, и собирает эти оценки силами сообщества — как карта, открываемая по ссылке в браузере без установки.

## 2. Primary user

Человек с инвалидностью в Абхазии, ищущий доступное место с телефона в браузере (часто с нарушением цветовосприятия или слабым зрением), без регистрации и без установки приложения. Вторичный — волонтёр-аудитор в поле (его инструмент тоже обязан быть доступным).

## 3. Principles (в порядке разрешения конфликтов — при столкновении побеждает верхний)

1. **Доступность стоит ПЕРЕД экраном, не после.** Приложение о доступности обязано быть доступным само. WCAG 2.2 AA — пол, не потолок. Никогда не отгружаем без: статус словом+цветом (не только цветом), контраст AA в обеих темах, зоны нажатия ≥44×44 px, alt-тексты, полная клавиатура + видимый фокус, корректные ARIA, `prefers-reduced-motion`. Этот принцип бьёт любой другой.
2. **Приложение — это карта, не мини-сайт.** Карта всегда фон; все панели — оверлеи поверх живой карты. Каждая «дверь» ведёт ко всей нужной инфо. Тап по пину → сразу карточка (bottom-sheet), без промежуточного попапа.
3. **Самый короткий путь.** Задачу можно решить в один клик — один клик. Второй клик добавляем только если этого требует логика, никогда «для красоты» или «чтобы выглядело солиднее».
4. **Не удивлять — незаметный интерфейс есть высшая форма.** Тёплый · надёжный · спокойный · лаконичный. Цвет несёт смысл только у статусов и маркеров; остальной интерфейс — спокойный нейтрал. Не игривый, не кричащий, не сухо-корпоративный, не перегруженный.
5. **Абхазский — приоритет читаемости; шрифт локальный.** Шрифт обязан цельно нести абхазский глиф-сет (Ԥ ԥ Ҭ ҭ Ҵ ҵ Ҷ ҷ Ҩ ҩ Ә ә Ҟ ҟ и т.д.) наравне с русским/латиницей. Файлы шрифтов — локально, не с CDN (при блокировке/VPN внешний шрифт обрежется, абхазский посыплется первым). Язык `ab` — первый в переключателе.

## 4. Success metric

Пользователь понимает, доступно ли конкретное место, за секунды после тапа по пину — без регистрации и без чтения инструкции. Просмотр всегда свободен; барьер (почта → код) появляется только на действии «добавить».

## 5. Out of scope

- Публичных комментариев / рейтингов / лайков — НЕТ. «Звёздность» = статус доступности (воспроизводимый, не субъективный отзыв).
- Геймификации — НЕТ (баллы / XP / лидерборды / бейджи). Уровни как роли — остаются.
- Соцсети ВНУТРИ продукта — НЕТ. Сообщество живёт в Telegram, новости в Instagram; в интерфейсе они только ссылки наружу.
- Не мини-сайт: не наполнять экран информацией сверх карты и её оверлеев.
- Движение пока не делаем — статика раньше анимации (см. DESIGN §10).
- Экспорт в OSM / Google / Яндекс — отдельный Research-трек, не этот surface (в модель данных теги OSM закладываем сразу).
